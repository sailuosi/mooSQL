## 测试辅助类

### TestDatabaseHelper
提供静态方法创建测试用的数据库实例：
- `CreateTestDBInstance()`: 创建 DBInstance
- `CreateSQLBuilder()`: 创建 SQLBuilder
- `CreateSQLClip()`: 创建 SQLClip

支持多种数据库类型（默认 SQLite，内存数据库）。

### TestEntity
提供测试用的实体类：
- `TestUser`: 用户实体（id, name, email, age, created_at, is_active）
- `TestOrder`: 订单实体（id, user_id, order_no, amount, created_at）

## 运行测试

### Visual Studio
1. 打开测试资源管理器（Test → Test Explorer）
2. 运行所有测试或选择特定测试
3. 查看测试结果和代码覆盖率

### 命令行
```bash
# 运行所有测试
dotnet test

# 运行特定测试类
dotnet test --filter "FullyQualifiedName~SQLBuilderTests"

# 生成代码覆盖率报告
dotnet test /p:CollectCoverage=true /p:CoverletOutputFormat=opencover
```

## 测试特点

1. **不依赖真实数据库**: 使用内存数据库或模拟连接
2. **SQL 验证**: 主要验证 SQL 语句构建的正确性
3. **完整覆盖**: 覆盖核心功能的常用场景和边界情况
4. **易于扩展**: 可以轻松添加新的测试用例

## 下一步改进建议

1. **增加边界测试**: 添加更多边界情况和异常场景测试
2. **性能测试**: 添加性能基准测试
3. **集成测试**: 添加需要真实数据库连接的集成测试
4. **代码覆盖率**: 目标达到 80% 以上覆盖率
5. **测试更多类**: 扩展到其他核心类（Repository、UnitOfWork 等）

## 注意事项

- 某些测试可能需要实体类正确配置了映射属性（SooTable、SooColumn）
- SQL 语句的具体格式可能因数据库方言而异
- 测试主要验证 SQL 构建逻辑，不验证 SQL 执行结果

## 贡献指南

添加新测试时：
1. 遵循 AAA 模式（Arrange-Act-Assert）
2. 使用 FluentAssertions 进行断言
3. 为测试方法添加清晰的注释
4. 确保测试独立且可重复执行
