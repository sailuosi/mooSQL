# 分表功能使用指南

> 面向 **使用 mooSQL 的应用开发人员**。介绍如何为实体配置分表、如何正确读写分表数据。  
> 框架内部设计与源码索引见 [分表功能设计与实现](./分表功能设计与实现.md)。

---

## 1. 什么时候用分表

适合分表的场景：

- 按**时间**增长的日志、订单、流水、审计记录
- 单表数据量过大，需要按月/按日拆物理表
- 历史数据归档：老表只读，新表继续写入

**不需要分表时**：不要配置 `ShardMode`，实体行为与普通表完全一致。

---

## 2. 快速开始（按月分表）

### 2.1 定义实体

```csharp
[SooTable("OrderLog_{year}{month}", ShardMode = TableShardMode.Month, ShardAnchor = "2024-1-1")]
public class OrderLog
{
    [SooColumn(IsPrimaryKey = true)]
    public long Id { get; set; }

    /// <summary>分片键：写入时据此路由物理表</summary>
    [SooColumn(Shard = true)]
    public DateTime CreateTime { get; set; }

    [SooColumn]
    public string Content { get; set; }
}
```

要点：

- `SooTable.Name` 中的 `{year}{month}` 等占位符决定物理表名格式
- **必须**用 `[SooColumn(Shard = true)]` 标记分片键（通常是时间字段）
- `ShardAnchor` 表示分表起始时间，影响「全部表」枚举与 DDL 同步范围

### 2.2 写入

```csharp
var repo = db.useRepo<OrderLog>();

// 单条插入：自动路由到 OrderLog_202406
repo.Insert(new OrderLog
{
    CreateTime = DateTime.Now,
    Content = "下单成功"
});

// 批量插入：内部按物理表分组，无需手动拆分
repo.InsertRange(logList);
```

写入前请确保：

1. 实体上 **CreateTime（分片键）已赋值**
2. 目标物理表已在数据库中存在（或开启自动建表，见第 6 节）

### 2.3 按 ID 查询单条（指定分表）

分表后 `GetById` **不会**自动搜索所有表，需告知框架查哪张物理表：

```csharp
// 方式一：仓储指定时间点
var log = repo.ForShard(new DateTime(2024, 6, 15)).GetById(1001L);

// 方式二：ShardScope（适合无实体行的查询上下文）
using (ShardScope.For<OrderLog>(new DateTime(2024, 6, 15)))
{
    log = repo.GetById(1001L);
}

// 方式三：直接指定表名
log = repo.UseTable("OrderLog_202406").GetById(1001L);
```

## 3. 分表模式说明

通过 `[SooTable(ShardMode = ...)]` 选择：

| ShardMode | 说明 | 物理表示例 |
|-----------|------|------------|
| `None` | 不分表（默认） | `OrderLog` |
| `Year` | 按年 | `OrderLog_2024` |
| `Quarter` | 按季度 | `OrderLog_2024Q2` |
| `Month` | 按月 | `OrderLog_202406` |
| `Week` | 按周 | `OrderLog_2024W24` |
| `Day` | 按日 | `OrderLog_20240615` |
| `Interval` | 固定间隔 | 见下节 |
| `Custom` | 自定义策略 | 由代码决定 |

### 3.1 间隔分表（Interval）

```csharp
[SooTable("Metric_{year}{month}", ShardMode = TableShardMode.Interval,
    ShardAnchor = "2024-1-1", ShardIntervalValue = 2, ShardIntervalUnit = "month")]
public class MetricLog
{
    [SooColumn(Shard = true)]
    public DateTime SampleTime { get; set; }
}
```

`ShardIntervalUnit` 支持：`year` / `month`（默认）/ `day` / `hour`。

### 3.2 自定义表名（Lambda）

不按时间周期、而是按业务规则（如用户 ID 取模）分表：

```csharp
// 启动时注册（MooClient 或 BaseClientBuilder）
client.useShard<OrderLog>(o => $"OrderLog_{o.CreateTime:yyyyMM}");

// 或按 ID 哈希
client.useShard<UserData>(u => $"user_data_{u.Id % 8}");
```

Lambda 方式适合**单点路由**（写入、按实体查单表）。跨表范围查询需自行指定表或使用自定义 `ITableShardStrategy`。

## 4. 跨表查询

默认 `GetList()` **只查一张表**（或无 ShardScope 时可能落到模板名）。跨时间段查询须用专用 API。

### 4.1 仓储 QueryRange

```csharp
var from = new DateTime(2024, 4, 1);
var to   = new DateTime(2024, 6, 30);

var list = repo.QueryRange(from, to, q =>
{
    q.where("Status", 1);
    q.orderby("CreateTime desc");
});
```

内部对 `OrderLog_202404`、`OrderLog_202405`、`OrderLog_202406` 做 `UNION ALL` 后返回合并结果。

### 4.2 SQLBuilder

```csharp
// 指定时间范围
var list = db.useSQL()
    .fromShardRange<OrderLog>(from, to)
    .where("Status", 1)
    .query<OrderLog>()
    .ToList();

// 链式分表（对标 SqlSugar SplitTable）
var list2 = db.useSQL()
    .splitTable<OrderLog>(from, to)
    .select("*")
    .where("Status", 1)
    .buildShardFrom()
    .query<OrderLog>()
    .ToList();

// 查最近 3 张分表（默认）
var recent = db.useSQL()
    .splitTable<OrderLog>()
    .takeRecent(3)
    .select("*")
    .buildShardFrom()
    .query<OrderLog>()
    .ToList();
```

### 4.3 SQLClip

```csharp
var clip = db.useClip();
clip.fromShardRange<OrderLog>(from, to, out var alias);
var list = clip.where(c => c.Status == 1).toList<OrderLog>();
```

### 4.4 查询最近 N 张表

未指定时间范围时，默认命中**最近 3 张**分表（可在表特性上调整）：

```csharp
[SooTable("...", ShardMode = TableShardMode.Month, DefaultRecentTables = 5)]
```

## 5. 更新与删除

更新、删除与插入一样，按**实体分片键**或**显式指定表**路由：

```csharp
var entity = new OrderLog { Id = 1, CreateTime = new DateTime(2024, 6, 1), Content = "更新" };
repo.Update(entity);  // → OrderLog_202406

// 或
repo.ForShard(new DateTime(2024, 6, 1)).DeleteById(1L);
```

**注意**：若分片键时间变更导致目标表变化，框架不会自动「迁表」；业务侧应先删旧表再插新表，或使用显式 `UseTable`。

---

## 6. 建表与 DDL

分表实体每张物理表结构相同，需要提前创建或在首次写入时自动创建。

### 6.1 手动 / 脚本建表

按命名规则预先创建：`OrderLog_202401`、`OrderLog_202402` …

### 6.2 批量同步结构

```csharp
// 将实体结构同步到指定时间范围内的所有分表
ShardDdlHelper.InitShardTables<OrderLog>(db, new DateTime(2024, 1, 1), DateTime.Now);
```

### 6.3 插入时自动建表

```csharp
[SooTable("OrderLog_{year}{month}", ShardMode = TableShardMode.Month,
    AutoCreateOnInsert = true)]
```

开启后，首次向某物理表插入时若表不存在，会调用 `createAuto` 建表。生产环境建议评估权限与并发后再启用。

---

## 7. 启动配置（可选）

特性声明之外，可在 `MooClient` / `BaseClientBuilder` 上编程式配置：

```csharp
var client = new MooClient();

// Lambda 分表
client.useShard<OrderLog>(o => $"OrderLog_{o.CreateTime:yyyyMM}");

// 修改分表行为
client.configureShard<OrderLog>(cfg =>
{
    cfg.AutoCreateOnInsert = true;
    cfg.DefaultRecentTables = 5;
    cfg.MaxTablesPerQuery = 12;
});

// 完整自定义策略（支持范围查询）
client.useShardStrategy<AuditLog>(new MyAuditShardStrategy());
```

`BaseClientBuilder` 提供同名链式方法：`useShard` / `configureShard` / `useShardStrategy`。

## 8. 常用 API 速查

| 场景 | 推荐 API |
|------|----------|
| 单条写入 | `repo.Insert(entity)` |
| 批量写入 | `repo.InsertRange(list)` |
| 按 ID 查单表 | `repo.ForShard(time).GetById(id)` |
| 无实体行查单表 | `using (ShardScope.For<T>(time)) { ... }` |
| 指定物理表 | `repo.UseTable("OrderLog_202406")` |
| 时间范围查询 | `repo.QueryRange(from, to, q => ...)` |
| SQLBuilder 范围 | `fromShardRange<T>(from, to)` |
| 最近 N 表 | `splitTable<T>().takeRecent(n).buildShardFrom()` |
| 枚举所有分表 | `ctx.GetShardHelper<T>().GetTables()` |
| 批量按表分组 | `new ShardTableHelper(en).GroupByTable(list)` |
| 同步 DDL | `ShardDdlHelper.InitShardTables<T>(db, from, to)` |

## 9. 使用注意与最佳实践

### 9.1 必须知道的行为

1. **Opt-in**：未配置 `ShardMode` 的实体不受任何分表逻辑影响。
2. **查询不自动跨表**：`GetList` / `GetById` 不会扫描所有分表，跨表须用 `QueryRange` 等 API。
3. **分片键要有值**：写入时分片键为空会导致路由失败或落到默认表名。
4. **全局 setTable 优先**：`EntityTranslator.setTable` 或 `UseTable` 会覆盖自动路由。

### 9.2 推荐做法

- 分片键统一用 **UTC 或固定时区** 的 `DateTime`，避免跨月边界歧义
- 跨月报表、统计一律走 `QueryRange`，并限定合理时间窗口
- 用 `MaxTablesPerQuery` 限制单次 UNION 表数量，防止 SQL 过大
- 定期对历史分表做归档/只读，新月份表提前 `InitShardTables` 建好

### 9.3 常见错误

| 现象 | 原因 | 处理 |
|------|------|------|
| 启动报「未标记分片键」 | 有 `ShardMode` 但未 `[SooColumn(Shard=true)]` | 补充分片键特性 |
| GetById 查不到 | 未指定分表 | 使用 `ForShard` / `ShardScope` |
| 列表数据不全 | 用了普通 GetList | 改用 `QueryRange` |
| 表不存在 | 物理表未创建 | DDL 同步或 `AutoCreateOnInsert` |
| 写入到错误表 | 分片键时间不对 | 检查 CreateTime 赋值 |

## 10. 完整示例

```csharp
// ── 实体 ──
[SooTable("AppLog_{year}{month}", ShardMode = TableShardMode.Month, ShardAnchor = "2024-1-1")]
public class AppLog
{
    [SooColumn(IsPrimaryKey = true, IsIdentity = true)]
    public long Id { get; set; }

    [SooColumn(Shard = true)]
    public DateTime LogTime { get; set; }

    [SooColumn]
    public string Message { get; set; }
}

// ── 初始化 DDL（可选，部署时执行一次）──
ShardDdlHelper.InitShardTables<AppLog>(db, new DateTime(2024, 1, 1), DateTime.Now.AddMonths(1));

// ── 业务代码 ──
var repo = db.useRepo<AppLog>();

// 写入
repo.Insert(new AppLog { LogTime = DateTime.Now, Message = "服务启动" });

// 查某天某条
var one = repo.ForShard(DateTime.Today).GetById(42L);

// 查本季度
var qStart = new DateTime(DateTime.Today.Year, (DateTime.Today.Month - 1) / 3 * 3 + 1, 1);
var qEnd = qStart.AddMonths(3).AddDays(-1);
var logs = repo.QueryRange(qStart, qEnd, q => q.where("Message", "like", "%错误%"));
```
