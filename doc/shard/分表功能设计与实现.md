# 分表功能设计与实现

> 面向 **mooSQL 项目开发人员**。说明分表（Shard）模块的设计思路、实现方式与功能边界。  
> 应用侧用法见 [分表功能使用指南](./分表功能使用指南.md)。

---

## 1. 背景与目标

### 1.1 业务场景

日志、订单、审计等数据量随时间线性增长，单表过大后带来索引膨胀、备份困难、DDL 锁表等问题。分表将同一逻辑实体映射到多张物理表，按分片键（通常是时间）路由读写。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **Opt-in** | 未配置分表的实体行为与改造前完全一致，零干扰 |
| **声明式 + 编程式** | 支持 `[SooTable]` / `[SooColumn]` 特性，也支持 `useShard` / `configureShard` |
| **写入自动路由** | Insert / Update / Delete 按实体分片键解析物理表名 |
| **查询显式跨表** | 默认 `GetList` / `GetById` 不自动 UNION，跨表须调用专用 API |
| **多入口一致** | Repository、SQLBuilder、SQLClip 共用同一套策略与表名解析 |
| **可扩展** | `ITableShardStrategy` 支持自定义路由（租户、哈希等） |

### 1.3 对标参考

| mooSQL | SqlSugar | FreeSql |
|--------|----------|---------|
| `[SooColumn(Shard = true)]` | `[SplitField]` | 分表字段 |
| `ShardMode` / `useShard` | `SplitType` | `AsTable` |
| `splitTable` / `QueryRange` | `SplitTable()` | 分表查询 |
| `ShardTableHelper.GroupByTable` | `SplitHelper` | — |

## 2. 核心概念

```
逻辑实体 (EntityInfo)
    ├── DbTableName          模板名，如 Order_{year}{month}
    ├── LiveName             是否动态表名
    ├── Shard (EntityShardConfig)
    │     ├── Mode           分表周期 / Custom / Interval
    │     ├── ShardKeyProperty / ShardKeyColumn
    │     ├── NameTemplate / TablePrefix / Anchor
    │     └── ResolveStrategy() → ITableShardStrategy
    └── NameParses["@shard"] → TableShardInterceptor
```

| 概念 | 含义 |
|------|------|
| **分片键** | 实体上 `[SooColumn(Shard = true)]` 标记的属性，写入/单点路由时从中取时间 |
| **物理表名** | 策略 `ResolvePoint` 输出的真实表名，如 `Order_202406` |
| **LiveName** | 实体启用动态表名解析；分表实体自动设为 `true` |
| **NameParses** | 表名拦截器字典；分表默认注册键 `@shard` |

## 3. 架构与数据流

### 3.1 模块位置

源码目录：`pure/src/ado/SQL/DBmodel/shard/`

| 文件 | 职责 |
|------|------|
| `ShardRegistration` | 特性解析、`useShard` 注册、拦截器挂载、`FinalizeEntityShard` |
| `EntityShardConfig` | 运行时分表配置，由特性或编程式 API 填充 |
| `ITableShardStrategy` | 策略接口：`ResolvePoint` / `ResolveRange` / `ResolveAllTables` |
| `TimeTableShardStrategy` | 年/季/月/周/日周期分表 |
| `IntervalTableShardStrategy` | 固定间隔分表（月/日/小时等） |
| `TableShardNameFormatter` | 模板占位符替换（`{year}` `{month}` 等） |
| `TableShardInterceptor` | 将策略适配为 `ITableNameInterceptor` |
| `ShardKeyHelper` | 从实体行提取分片键时间值 |
| `ShardScope` | 异步上下文单点时间点（无实体行时的查询路由） |
| `ShardQueryBuilder` | 解析目标表列表、构建 UNION ALL |
| `ShardQueryOptions` | 范围 / 最近 N 表 / 指定表 / 过滤 |
| `ShardTableHelper` | 批量分组、表名枚举（对标 SplitHelper） |
| `ShardDdlHelper` | 分表 DDL 同步、插入前自动建表 |
| `ShardJoinHelper` | 分表侧子查询 SQL 构建（JOIN 辅助，Phase D） |
| `ShardSplitContext` | SQLBuilder 分表链状态 |

扩展入口：

| 文件 | 职责 |
|------|------|
| `MooClient.shard.cs` | `useShard` / `useShardStrategy` / `configureShard` |
| `BaseClientBuilder.cs` | Builder 链式转发 |
| `EntityTranslator.shard.cs` | `GetResolvedTableName`、`BuildSelectFromRange` |
| `SooRepository.shard.cs` | `ForShard` / `UseTable` / `QueryRange` |
| `SQLBuilderShardExtensions.cs` | `splitTable` / `fromShardRange` / `buildShardFrom` |
| `SQLClip.shard.cs` | `fromShardRange` |
| `MooEntityAnalyser.cs` / `InternalEntityParser.cs` | 解析 `SooColumn.Shard` |

### 3.2 实体解析流程

```
类型加载
  → MooEntityAnalyser.ReadEntityAttr(SooTableAttribute)
       → ShardRegistration.ApplyTableAttribute
  → ParseColumn 遍历 SooColumnAttribute
       → ca.Shard == true → ShardRegistration.MarkShardField
  → ParseEntity 结束
       → ShardRegistration.FinalizeEntityShard
            → 校验分片键（非 Custom 模式必填）
            → LiveName = true
            → RegisterInterceptor → NameParses["@shard"]
```

**校验规则**：`ShardMode != Custom` 且未标记 `SooColumn(Shard = true)` 时，在 `FinalizeEntityShard` 抛出 `InvalidOperationException`。

### 3.3 表名解析优先级

`EntityTranslator.resolveTableName` 决策顺序：

```
1. loadName 回调（仓储 tbname / ForShard / UseTable 等显式指定）
2. parseTableName：
   a. EntityTranslator 全局 _onParseTableName 回调
   b. ShardScope.Current（仅 row == null 且无 loadName 时）
   c. LiveName + NameParses 拦截器链（含 TableShardInterceptor）
3. DbTableName（静态表名兜底）
```

写入路径（Insert/Update/Delete）传入实体行 `row`，`TableShardInterceptor.Parse` 从分片键取时间后 `ResolvePoint`。

查询路径（无实体行）可依赖 `ShardScope.For<T>(point)` 或显式 `ForShard` / `UseTable`。

### 3.4 写入与批量

```
Insert(entity)
  → insertInner
  → Translator.GetResolvedTableName(En, entity, loader)
  → TableShardInterceptor.Parse(entity) → 物理表名
  → INSERT INTO {物理表} ...

InsertRange(entities)  [分表实体]
  → ShardTableHelper.GroupByTable
  → 按物理表分组后逐组 insertInner

AutoCreateOnInsert == true
  → ShardDdlHelper.EnsureTableForInsert（表不存在则 createAuto）
```

### 3.5 跨表查询

```
QueryRange / fromShardRange / splitTable + buildShardFrom
  → ShardQueryBuilder.ResolveTables(en, options)
  → strategy.ResolveRange(from, to) 或 Recent(n) 等
  → BuildUnionFrom：多表 SELECT ... UNION ALL
```

**设计约束**：普通 `GetList()` / `GetById()` **不会**自动展开为多表 UNION，避免无意全表扫描。

## 4. 策略实现

### 4.1 ITableShardStrategy

```csharp
public interface ITableShardStrategy
{
    string ResolvePoint(EntityInfo en, object rowOrNull, DateTime? pointTime);
    IReadOnlyList<string> ResolveRange(EntityInfo en, DateTime from, DateTime to);
    IReadOnlyList<string> ResolveAllTables(EntityInfo en);
}
```

- **ResolvePoint**：单条写入/单表查询的目标表名
- **ResolveRange**：时间范围内涉及的全部物理表（有序、去重）
- **ResolveAllTables**：从 Anchor 到当前时间的全部分表（DDL 同步等）

### 4.2 TimeTableShardStrategy

支持 `TableShardMode`：`Year` / `Quarter` / `Month` / `Week` / `Day`。

- 按模式枚举时间桶，对每个桶调用 `TableShardNameFormatter.Format`
- `MaxTablesPerQuery` 可截断范围结果

### 4.3 IntervalTableShardStrategy

`ShardMode = Interval` 时使用。从 `ShardAnchor` 起按 `ShardIntervalValue` + `ShardIntervalUnit`（`year`/`month`/`day`/`hour`）步进。

### 4.4 Custom 模式

两种方式：

1. **Lambda**：`client.useShard<T>(row => tableName)` → `FuncTableNameCepter`，仅实现单点路由，范围查询需自定义策略
2. **完整策略**：`useShardStrategy<T>(ITableShardStrategy)` 或 `[SooTable(CustomShardStrategyType = typeof(...))]`

`EntityShardConfig.ResolveStrategy()` 选择顺序：  
`CustomStrategy` → `CustomShardStrategyType` 实例化 → `Interval` → 时间策略 → `null`

### 4.5 表名模板

`TableShardNameFormatter` 支持的占位符：

| 占位符 | 示例 |
|--------|------|
| `{prefix}` | 表前缀 |
| `{year}` | 2024 |
| `{month}` | 06 |
| `{day}` | 15 |
| `{quarter}` | 2 |
| `{week}` | ISO 周序号 |

默认模板（未指定 `NameTemplate` 时）：

| Mode | 默认模板 |
|------|----------|
| Year | `{prefix}_{year}` |
| Quarter | `{prefix}_{year}Q{quarter}` |
| Month | `{prefix}_{year}{month}` |
| Week | `{prefix}_{year}W{week}` |
| Day | `{prefix}_{year}{month}{day}` |

`SooTable.Name` 中含 `{` 时，前缀取 `{` 之前部分（去尾 `_`）。

## 5. 特性与配置映射

### 5.1 SooTableAttribute → EntityShardConfig

| SooTable 属性 | EntityShardConfig 字段 |
|---------------|------------------------|
| `ShardMode` | `Mode` |
| `Name` / `NameTemplate` | `NameTemplate` / `TablePrefix` |
| `ShardAnchor` | `Anchor` |
| `ShardIntervalValue` | `IntervalValue` |
| `ShardIntervalUnit` | `IntervalUnit` |
| `ShardFirstSpan` | `FirstSpan` |
| `DefaultRecentTables` | `DefaultRecentTables`（默认 3） |
| `MaxTablesPerQuery` | `MaxTablesPerQuery` |
| `AutoCreateOnInsert` | `AutoCreateOnInsert` |
| `CustomShardStrategyType` | `CustomShardStrategyType` |
| `LiveName` | `EntityInfo.LiveName` |

### 5.2 SooColumnAttribute

| 属性 | 作用 |
|------|------|
| `Shard = true` | 标记分片键；填充 `ShardKeyProperty` / `ShardKeyColumn` |

同一实体多个 `Shard = true` 时，**首个**解析到的属性生效。

### 5.3 ShardQueryOptions

| 字段 | 用途 |
|------|------|
| `RangeFrom` / `RangeTo` | 时间范围 |
| `TakeRecent` | 最近 N 张表（无范围时的默认行为） |
| `InTables` | 显式指定表名列表 |
| `AllTables` | 全部已配置分表 |
| `TableFilter` | 表名过滤谓词 |

解析逻辑见 `ShardQueryBuilder.ResolveTables`：范围 > 指定表 > Recent > 兜底 `DbTableName`。

## 6. 零干扰约束（必须遵守）

1. **禁止**在默认 CRUD（`GetList` / `GetById` / 普通 `select().from<T>()`）中无条件调用 `splitTable` / `ResolveRange`。
2. `ShardScope` 仅在 `en.Shard != null` 或 `LiveName == true` 时参与解析。
3. `TableShardMode.None` 必须为枚举值 `0`（默认值 = 不分表）。
4. 全局 `EntityTranslator.setTable` / 仓储 `UseTable` 仍覆盖分表路由（与改造前一致）。
5. 未配置分表的实体：`EntityInfo.Shard == null`，所有分表 API 退化为单表行为。

## 9. 扩展指南

### 9.1 新增分表模式

1. 在 `TableShardMode` 增加枚举值（保持 `None = 0`）
2. 在 `EntityShardConfig.ResolveStrategy()` 中分支
3. 实现 `ITableShardStrategy` 或在 `TimeTableShardStrategy.EnumerateBuckets` 扩展
4. 补充 `TableShardNameFormatter.DefaultTemplate`
5. 增加 `ShardTableTests` 用例

### 9.2 自定义策略示例骨架

```csharp
public sealed class TenantShardStrategy : ITableShardStrategy
{
    public string ResolvePoint(EntityInfo en, object rowOrNull, DateTime? pointTime)
    {
        var tenantId = (rowOrNull as ITenant)?.TenantId ?? "default";
        return $"{en.Shard.TablePrefix}_{tenantId}";
    }

    public IReadOnlyList<string> ResolveRange(EntityInfo en, DateTime from, DateTime to)
        => throw new NotSupportedException("请通过 InTables 或自定义查询指定表");

    public IReadOnlyList<string> ResolveAllTables(EntityInfo en)
        => throw new NotSupportedException();
}
```

### 9.3 注意事项

- 分片键目前仅支持 `DateTime` / `DateTimeOffset` / 可解析的 `string`（见 `ShardKeyHelper.ToDateTime`）
- `ResolveRange` 在 Interval 模式有 500 表上限保护
- `ShardScope` 在 .NET Framework 使用 `[ThreadStatic]`，在 .NET 6+ 使用 `AsyncLocal`
