# mooSQL.Pure 项目功能介绍和技术架构分析

## 一、项目概述

mooSQL.Pure 是一个纯粹、高可用、自由的面向数据库的连接库，支持多种数据库类型，提供了丰富的数据库操作功能。项目采用 C# 开发，支持 .NET Framework 4.5.1/4.6.2 以及 .NET 6.0/8.0/10.0 等多个框架版本。

### 1.1 项目定位

- **纯粹性**：专注于数据库访问，不依赖第三方 ORM 框架
- **高可用性**：支持主从复制、连接池、事务管理等企业级特性
- **自由度**：提供灵活的 SQL 构建、LINQ 查询、仓储模式等多种使用方式

### 1.2 支持的数据库类型

- MySQL / OceanBase
- Microsoft SQL Server
- Oracle
- PostgreSQL
- Taos
- GBase8a
- SQLite
- Oscar

## 二、核心功能模块

### 2.1 ADO.NET 核心层 (ado)

#### 2.1.1 SQL 构建器 (builder)
- **SQLBuilder**: 核心 SQL 构建器，支持链式调用
  - 支持 SELECT、INSERT、UPDATE、DELETE、MERGE INTO 等操作
  - 支持条件构建（WHERE）、排序（ORDER BY）、分页（PAGING）
  - 支持事务管理
  - 支持 SQL 注入防护
  - 支持参数化查询
  - 支持 CTE（公用表表达式）和 UNION 查询

- **SQLBuilderDymatic**: 动态 SQL 构建
- **SQLBuilderSelect/Where/Save**: 特定操作的专用构建器

#### 2.1.2 数据访问层 (data)
- **DBInstance**: 数据库实例管理，封装连接和方言信息
- **DBInsCash**: 数据库配置缓存，管理多个数据库连接
- **DBExecutor**: 数据库执行器，负责 SQL 执行和事务管理
- **DBRunner**: 数据库运行器，支持批量 SQL 执行
- **CmdExecutor**: 命令执行器，封装 SqlCommand 功能

#### 2.1.3 方言系统 (dialect)
- **Dialect**: 数据库方言抽象基类
- **DialectFactory**: 方言工厂，支持多种数据库方言注册和获取
- 每种数据库都有对应的方言实现，处理数据库特定的 SQL 语法差异

#### 2.1.4 缓存系统 (cache)
- **ISooCache**: 缓存接口
- **HashCache**: 基于哈希表的缓存实现
- 支持绝对过期、相对过期、永不过期三种策略
- 支持条件删除、批量删除等操作

#### 2.1.5 类型处理 (typeHandle)
- 处理不同数据库类型之间的转换
- 支持自定义类型处理器

#### 2.1.6 SQL 结构定义 (SQL)
- 完整的 SQL 语法树定义
- 支持 SELECT、INSERT、UPDATE、DELETE、DDL 等语句的结构化表示
- 支持表达式访问器模式进行 SQL 生成

### 2.2 ADO 扩展层 (adoext)

#### 2.2.1 仓储模式 (repository)
- **SooRepository<T>**: 通用仓储实现
  - 提供 CRUD 操作的统一接口
  - 支持按 ID 查询、批量操作
  - 支持分页查询、树形结构查询
  - 支持递归查询（最多 50 层）
  - 提供保存前/后的钩子方法

- **SooUnitOfWork**: 工作单元模式，支持事务管理

#### 2.2.2 SQL 片段 (clip)
- **SQLClip**: 强类型 SQL 片段构建器
- 支持 Lambda 表达式构建查询条件
- 提供类型安全的查询方式

#### 2.2.3 导航查询 (nav)
- 支持实体间的导航关系查询

### 2.3 LINQ 支持 (linq)

#### 2.3.1 查询提供器
- **EntityQueryProvider**: 实现 IQueryProvider 接口
- **EntityQueryable<T>**: 实现 IQueryable<T> 接口
- 支持标准 LINQ 操作：Where、Select、OrderBy、GroupBy 等

#### 2.3.2 表达式翻译
- **BaseTranslateVisitor**: 表达式访问器基类
- **FastExpressionTranslatVisitor**: 快速表达式翻译器
- **MethodVisitor**: 方法调用访问器
- 将 LINQ 表达式树转换为 SQL 语句

#### 2.3.3 查询编译器
- **BaseQueryCompiler**: 查询编译器基类
- **FastLinqCompiler**: 快速 LINQ 编译器
- 支持编译查询缓存，提升性能

### 2.4 认证授权 (auth)

#### 2.4.1 权限控制
- **AuthorBuilder**: 权限构建器
- **AuthContext**: 认证上下文
- 支持数据级别的权限控制
- 支持字段级别的读写权限控制
- 支持自定义权限方言

### 2.5 配置管理 (config)

#### 2.5.1 数据库配置
- **DBBuilder**: 数据库构建器
- **DBPosition**: 数据库位置管理
- **FastConfigExtensions**: 快速配置扩展方法
- 支持 XML 配置文件加载
- 支持代码配置

### 2.6 Excel 支持 (excel)

- 支持 Excel 文件的导入导出
- 支持数据批量导入数据库

### 2.7 主从复制 (slave)

- **ModifyMediator**: 修改中介器，管理主从复制
- **SlaveTeam**: 从库团队
- **SlaveCmdWorker**: 从库命令工作器
- 支持主库执行后自动同步到从库
- 支持信号机制，控制何时触发从库同步
- 支持队列机制，避免从库阻塞影响主库性能

### 2.8 扩展功能 (zoom)

- **AI**: AI 相关功能
- **FTP**: FTP 文件传输支持

## 三、技术架构分析

### 3.1 设计模式

#### 3.1.1 工厂模式 (Factory Pattern)
- **DialectFactory**: 方言工厂，根据数据库类型创建对应的方言实例
- **DBClientFactory**: 客户端工厂，创建各种客户端对象（SQLBuilder、Repository 等）
- **CacheFactory**: 缓存工厂，创建缓存实例

**优点**：
- 解耦对象创建和使用
- 便于扩展新的数据库类型或客户端类型

**实现示例**：
```csharp
public class DialectFactory : DialectFactoryBase
{
    public DialectFactory() {
        this.useDialect(DataBaseType.MySQL, () => new MySQLDialect());
        this.useDialect(DataBaseType.MSSQL, () => new MSSQLDialect());
        // ...
    }
}
```

#### 3.1.2 建造者模式 (Builder Pattern)
- **SQLBuilder**: SQL 语句构建器，支持链式调用
- **BaseClientBuilder**: 客户端构建器，配置和构建 MooClient

**优点**：
- 提供流畅的 API
- 支持复杂对象的逐步构建

**实现示例**：
```csharp
var sql = kit.setTable("users")
    .select("id", "name", "email")
    .where("age", ">", 18)
    .orderBy("name")
    .toSelect();
```

#### 3.1.3 仓储模式 (Repository Pattern)
- **SooRepository<T>**: 通用仓储实现
- **ISooRepository<T>**: 仓储接口

**优点**：
- 封装数据访问逻辑
- 提供统一的 CRUD 接口
- 便于单元测试

#### 3.1.4 工作单元模式 (Unit of Work Pattern)
- **SooUnitOfWork**: 工作单元实现
- 管理事务和多个仓储的协调

**优点**：
- 保证数据一致性
- 统一管理事务

#### 3.1.5 策略模式 (Strategy Pattern)
- **Dialect**: 不同数据库的 SQL 生成策略
- 每种数据库实现自己的方言策略

**优点**：
- 支持多种数据库类型
- 易于扩展新的数据库支持

#### 3.1.6 访问者模式 (Visitor Pattern)
- **ExpressionVisitor**: LINQ 表达式访问器
- **ClauseTranslateVisitor**: SQL 子句翻译访问器

**优点**：
- 将表达式树遍历和 SQL 生成分离
- 支持复杂的表达式转换

#### 3.1.7 中介者模式 (Mediator Pattern)
- **ModifyMediator**: 主从复制中介器
- 协调主库和从库之间的同步

**优点**：
- 解耦主库和从库
- 统一管理同步逻辑

#### 3.1.8 单例模式 (Singleton Pattern)
- **MooClient**: 客户端单例实例
- 通过 DBInsCash 管理客户端实例

### 3.2 功能架构

```
┌─────────────────────────────────────────────────────────┐
│                    应用层 (Application)                   │
│  Repository / LINQ / SQLBuilder / UnitOfWork            │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    AOP 层 (Aspect)                       │
│  MooClient / DBClientFactory / Events / Cache           │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                  ADO 核心层 (Core)                       │
│  DBInstance / DBExecutor / Dialect / SQLBuilder         │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                 数据库驱动层 (Driver)                     │
│  ADO.NET (SqlConnection / MySqlConnection / ...)        │
└─────────────────────────────────────────────────────────┘
```

#### 3.2.1 分层说明

1. **应用层**
   - 提供 Repository、LINQ、SQLBuilder 等高级 API
   - 业务代码直接使用这一层

2. **AOP 层**
   - 提供横切关注点：缓存、日志、事件、权限控制
   - MooClient 作为核心协调者

3. **ADO 核心层**
   - 数据库连接管理
   - SQL 构建和执行
   - 方言处理

4. **数据库驱动层**
   - 使用标准 ADO.NET 驱动
   - 不直接依赖第三方 ORM

### 3.3 核心组件关系

```
MooClient (核心协调者)
├── DialectFactory (方言工厂)
│   └── Dialect (方言实现)
├── DBClientFactory (客户端工厂)
│   ├── SQLBuilder
│   ├── SooRepository<T>
│   └── SooUnitOfWork
├── DBInsCash (数据库实例缓存)
│   └── DBInstance (数据库实例)
│       ├── DBExecutor (执行器)
│       └── Dialect (方言)
├── Cache (缓存)
├── Events (事件)
└── EntityContext (实体上下文)
```

## 四、优缺点分析

### 4.1 优点

#### 4.1.1 功能丰富
- ✅ 支持多种数据库类型（8+ 种）
- ✅ 提供多种使用方式：SQLBuilder、LINQ、Repository
- ✅ 支持主从复制、缓存、权限控制等企业级特性
- ✅ 支持 Excel 导入导出、FTP 等扩展功能

#### 4.1.2 架构设计良好
- ✅ 采用多种设计模式，代码结构清晰
- ✅ 分层明确，职责单一
- ✅ 支持扩展和自定义

#### 4.1.3 性能优化
- ✅ 支持查询编译缓存
- ✅ 支持连接池
- ✅ 支持结果缓存
- ✅ 支持批量操作

#### 4.1.4 易用性
- ✅ 提供流畅的链式 API
- ✅ 支持强类型的 LINQ 查询
- ✅ 支持仓储模式，简化 CRUD 操作
- ✅ 良好的错误处理和日志支持

#### 4.1.5 跨平台支持
- ✅ 支持多个 .NET 版本（.NET Framework 4.5.1+ 和 .NET 6.0+）
- ✅ 使用标准 ADO.NET，兼容性好

### 4.2 缺点

#### 4.2.1 代码复杂度
- ❌ 项目结构庞大，模块众多，学习曲线较陡
- ❌ 某些功能实现较为复杂（如 LINQ 翻译、主从复制）

#### 4.2.2 文档不足
- ❌ README 文件内容较少，且部分为乱码
- ❌ 缺少完整的 API 文档和使用示例
- ❌ 缺少架构设计文档

#### 4.2.3 测试覆盖
- ❌ 未看到单元测试项目
- ❌ 缺少集成测试

#### 4.2.4 性能问题
- ⚠️ 某些操作可能存在性能问题（如递归查询限制 50 层）
- ⚠️ 缓存实现相对简单，可能不适合高并发场景

#### 4.2.5 依赖管理
- ⚠️ 部分功能依赖 System.Runtime.Caching，不同框架版本需要不同版本
- ⚠️ 某些功能可能依赖外部库（如 Excel、FTP）

#### 4.2.6 错误处理
- ⚠️ 部分异常处理可能不够完善
- ⚠️ 错误信息可能不够友好

## 六、总结

mooSQL.Pure 是一个功能丰富、架构良好的数据库访问库。它提供了多种使用方式，支持多种数据库类型，并包含了许多企业级特性。然而，项目在文档、测试、性能优化等方面还有改进空间。

### 6.1 核心优势
1. **功能全面**：覆盖了数据库访问的各个方面
2. **架构清晰**：采用多种设计模式，代码结构良好
3. **易于扩展**：支持自定义方言、缓存、权限等
4. **跨平台**：支持多个 .NET 版本

### 6.2 主要挑战
1. **学习曲线**：项目庞大，需要时间理解
2. **文档不足**：缺少完整的文档和示例
3. **测试覆盖**：需要添加更多测试
4. **性能优化**：某些场景需要进一步优化

### 6.3 发展方向
1. **完善文档**：提供完整的 API 文档和使用指南
2. **增强测试**：添加单元测试和集成测试
3. **性能优化**：优化关键路径的性能
4. **模块化**：考虑拆分为多个包，提高灵活性
5. **现代化**：全面支持异步、依赖注入等现代特性
