
构建原则：

必有条件：
	1、词条解析必须在业务侧能够自定义行为，如本单位及下级这样词条的读取；
	2、词条的加载过滤、应用时刻都能够由业务侧自定义行为；

架构约束：
	1、方言（需要在业务侧个性化的特性和行为），需要统一引领+领域划分，既不杂揉，职责分明，易于理解和扩展。
	2、精简对外输出的成员。原则上业务侧主要继承 Builder对象，实现顶级方言对象 AuthDialect；
	3、泛型污染控制，为便于业务侧使用，仅在 builder下传递方言泛型。其它不在传递，使用行为约束。


编码时刻约束：
	1、除 AuthorBuilder外，其它类均不增加 方言泛型约束，由参数传递、行为控制等方式实现。
	2、在 各方言类中，均不得有对业务侧开放访问的固定成员。
